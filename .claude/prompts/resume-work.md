## 作業再開のプロンプト

このプロンプトを使用して、前回のセッションから作業を再開します。

```
前回の作業内容を確認し、続きから作業を再開しましょう。
.claude/summary/ にある前回のサマリーを参照して、次のステップを提案してください。
```

より具体的な指示が必要な場合：

```
.claude/summary/${file}.md を読み込み、前回の作業状況を確認してください。
次の実装ステップについて提案し、作業を再開しましょう。
```

### 効果的な作業再開のポイント

1. **コンテキストの再確認**
   - 前回までに達成した内容
   - 現在の実装状況と課題
   - 次に取り組むべきタスク

2. **焦点の明確化**
   - 今回のセッションで達成したい目標
   - 優先すべきタスクの特定
   - 時間配分の提案

3. **アプローチの選択**
   - 複数の実装方法がある場合の選択肢
   - 各アプローチのメリット・デメリット
   - 推奨される実装方法とその理由

4. **作業の進め方**
   - ファイル単位または機能単位での実装順序
   - テスト方法の提案
   - 作業完了の判断基準

### 追加のオプション

特定の観点から作業を進める場合：

```
前回の作業を ${perspective} の観点から見直し、続きの実装を提案してください。
```

`${perspective}` には、以下のような観点を指定できます：
- パフォーマンス最適化
- コード品質向上
- 保守性の改善
- テスト容易性
- ユーザビリティ
- 拡張性
